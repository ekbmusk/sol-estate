# CarbonKZ — Полный справочник проекта

Всё о проекте простым языком: что это, зачем, как работает, и что значат все термины.

---

## Что такое CarbonKZ

CarbonKZ — это онлайн-платформа (маркетплейс), где можно покупать и продавать **углеродные кредиты** казахстанских зелёных проектов. Работает на блокчейне Solana. Все сделки прозрачны, необратимы и доступны любому — не только крупным заводам.

**Одним предложением:** мы превращаем право на выброс CO₂ в цифровой токен, который можно купить, продать или сжечь как доказательство компенсации выбросов.

## Зачем это нужно

### Проблема Казахстана

| Факт | Цифра |
|------|-------|
| Выбросы CO₂ в год | 349 млн тонн |
| На одного человека | 17.5 тонн (в 3 раза больше, чем в среднем по ОЭСР) |
| Место в мире по CO₂ на единицу ВВП | 4-е |
| Доля угля в энергобалансе | ~50% |
| Текущая цена углеродного кредита | $1 за тонну |
| Целевая цена к 2030 году | $50 за тонну (рост в 50 раз) |
| Участников на бирже Caspy | 135 компаний |
| Блокчейн-проектов по углеродным кредитам в Центральной Азии | 0 |

### Почему рынок мёртв

- **Закрытый доступ** — торговать могут только крупные заводы с выбросами > 20 000 тонн/год. Банки, малый бизнес, частные инвесторы — не допущены.
- **Нулевая ликвидность** — всего 135 участников, торги приостанавливались в 2022.
- **Double counting** — один и тот же кредит заявляется несколькими компаниями. Нет механизма, чтобы это предотвратить.
- **Непрозрачность** — верификация проектов делается вручную, аудиторы могут иметь конфликт интересов.

### Почему сейчас самое время

- Правительство поставило цель: $50 за тонну к 2030, углеродная нейтральность к 2060.
- World Bank выделил $4.8 млн на развитие углеродного рынка КЗ (март 2025).
- Solana Economic Zone Kazakhstan — зона Web3 на $100 млн (май 2025).
- Нацбанк РК запустил KZTE стейблкоин на Solana (сентябрь 2025).
- ЦБ РК выделил $350 млн на цифровые активы (март 2026).

## Как всё работает (полный цикл)

```
1. СОЗДАНИЕ      Владелец зелёного проекта регистрирует его на платформе
        ↓
2. ВЕРИФИКАЦИЯ   Независимый верификатор проверяет документы и подтверждает проект
        ↓
3. ИНВЕСТИРОВАНИЕ  Любой человек покупает доли проекта за KZTE (тенге)
        ↓
4. ДОХОД         Загрязнители покупают углеродные кредиты → деньги идут в хранилище
        ↓
5. ДИВИДЕНДЫ     Инвесторы забирают свою долю дохода
        ↓
6. ГАШЕНИЕ       Загрязнитель «сжигает» кредиты → необратимое доказательство компенсации
```

### Пошагово

**Шаг 1 — Создание проекта (initialize_project)**
Владелец солнечной фермы или ветропарка заходит на платформу и регистрирует проект. Указывает: название, тип (солнечная/ветер/лес/промышленность), сколько тонн CO₂ в год сокращает, сколько долей доступно, цену за долю, хеш верификационного документа. Система создаёт два типа токенов: CarbonToken (собственно кредиты) и ShareToken (доли проекта), а также хранилище для денег (vault).

**Шаг 2 — Верификация (verify_project)**
Независимый эксперт проверяет реальность проекта. Он загружает хеш документа — если хеш совпадает с тем, что указал владелец, проект получает статус «верифицирован». Без верификации инвестировать нельзя.

**Шаг 3 — Инвестирование (invest)**
Обычный человек подключает крипто-кошелёк, выбирает проект и покупает доли. Деньги (KZTE) уходят в хранилище проекта. Взамен инвестор получает ShareToken — токены, подтверждающие его долю.

**Шаг 4 — Распределение дохода (distribute_revenue)**
Когда загрязняющие компании покупают углеродные кредиты проекта, владелец проекта вносит выручку в хранилище. Система автоматически рассчитывает, сколько дивидендов приходится на каждую долю.

**Шаг 5 — Получение дивидендов (claim_dividends)**
Инвестор нажимает «Получить дивиденды» — и KZTE переводятся из хранилища на его кошелёк. Получить одни и те же дивиденды дважды невозможно — система запоминает, что уже было выплачено.

**Шаг 6 — Гашение кредитов (retire_credits) — главная фича**
Компания-загрязнитель хочет компенсировать свои выбросы. Она «сжигает» (burn) углеродные токены — они исчезают навсегда. Одновременно создаётся неизменяемая запись в блокчейне: кто, сколько, когда и зачем погасил. Эту запись можно показать аудиторам. Сожжённый токен больше не существует — double counting невозможен.

## Словарь терминов

### Блокчейн и Solana

| Термин | Что это простым языком |
|--------|----------------------|
| **Блокчейн** | Цифровая книга записей, которую нельзя подделать. Каждая запись (транзакция) видна всем и хранится навечно. |
| **Solana** | Конкретный блокчейн. Быстрый (~400 мс на транзакцию), дешёвый (~$0.00025 за транзакцию), экологичный (PoS). |
| **Транзакция (tx)** | Одно действие в блокчейне: перевод денег, покупка доли, сжигание токена. Каждая транзакция получает уникальный ID (подпись), по которому её можно найти в Explorer. |
| **Кошелёк (wallet)** | Цифровой «счёт» пользователя в блокчейне. У каждого кошелька есть публичный адрес (как номер карты) и приватный ключ (как пин-код). |
| **Phantom / Solflare** | Приложения-кошельки для Solana. Устанавливаются как расширение в браузере. Через них пользователь подключается к платформе и подписывает транзакции. |
| **Devnet** | Тестовая сеть Solana. Все токены там ненастоящие — для разработки и демо. Mainnet — настоящая сеть с реальными деньгами. |
| **Explorer** | Сайт (explorer.solana.com), где можно посмотреть любую транзакцию, любой аккаунт, любой токен в блокчейне. Как выписка из банка, но публичная. |
| **Program (программа)** | Смарт-контракт на Solana. Код, который живёт в блокчейне и выполняется автоматически. Наша программа — `carbon_kz`. |
| **Program ID** | Уникальный адрес нашей программы в блокчейне: `3nLd8C3s2SAMVWXHy1vb7719zVPKPJWKrgxDDJ9pRRkg`. |
| **Instruction (инструкция)** | Одна конкретная команда, которую можно вызвать в программе. Например, `invest` — инвестировать, `retire_credits` — погасить кредиты. У нас 14 инструкций. |
| **Account (аккаунт)** | Место в блокчейне, где хранятся данные. Не путать с «аккаунтом пользователя» — это именно запись данных. Например, CarbonProject — аккаунт с информацией о проекте. |
| **PDA (Program Derived Address)** | Специальный адрес, который «принадлежит» программе, а не человеку. Генерируется из набора слов (seeds). Никто не может подделать этот адрес — его контролирует только программа. |
| **Seeds (сиды)** | Набор слов/данных, из которых вычисляется PDA-адрес. Например, для проекта сиды = `["project", "ses-yasavi"]`. Одни и те же сиды всегда дают один и тот же адрес. |
| **Bump** | Техническое число (0–255), которое добавляется к seeds, чтобы PDA-адрес был валидным. Вычисляется автоматически, хранится в аккаунте. |
| **SPL Token** | Стандарт токенов на Solana (аналог ERC-20 на Ethereum). Все наши токены (KZTE, CarbonToken, ShareToken) — SPL Token. |
| **Mint** | «Фабрика» токенов. Каждый тип токена имеет свой mint-адрес. Mint определяет, сколько токенов существует, сколько десятичных знаков и кто может создавать новые токены. |
| **Mint Authority** | Кто имеет право создавать (чеканить) новые токены. В нашем случае — PDA проекта (не человек!), чтобы никто не мог бесконтрольно печатать токены. |
| **ATA (Associated Token Account)** | Персональный «карман» для конкретного типа токена. У каждого кошелька для каждого токена есть свой ATA. Например: ATA вашего кошелька для KZTE, отдельный ATA для ShareToken. |
| **CPI (Cross-Program Invocation)** | Когда одна программа вызывает другую. Например, наша программа вызывает SPL Token программу, чтобы перевести KZTE. |
| **Lamports** | Минимальная единица на Solana (как копейки для рубля). 1 SOL = 1 000 000 000 lamports. Для KZTE: 1 KZTE = 1 000 000 units (6 decimals). |
| **Anchor** | Фреймворк для написания программ на Solana. Упрощает работу: автоматическая проверка аккаунтов, сериализация данных, генерация IDL. |
| **IDL (Interface Definition Language)** | JSON-файл, описывающий все инструкции и аккаунты программы. Генерируется автоматически при `anchor build`. Фронтенд использует IDL, чтобы знать, как общаться с программой. |
| **Rent** | «Арендная плата» за хранение данных в блокчейне Solana. Оплачивается при создании аккаунта. Если на аккаунте достаточно SOL (rent-exempt), плата не взимается. |
| **Solana Actions / Blinks** | Способ вызвать действие в блокчейне через URL или QR-код. Пользователь сканирует QR → открывается кошелёк → одобряет транзакцию. Не нужно заходить на сайт. |

### Углеродный рынок

| Термин | Что это простым языком |
|--------|----------------------|
| **Углеродный кредит (Carbon Credit)** | Право на выброс 1 тонны CO₂. Если завод выбросил меньше нормы, он может продать «лишние» разрешения. Если выбросил больше — обязан купить. |
| **KZ ETS (Kazakhstan Emissions Trading System)** | Казахстанская система торговли выбросами. Работает с 2013 года. 135 компаний с выбросами > 20 000 т/год обязаны участвовать. Биржа называется Caspy Exchange. |
| **Caspy Exchange** | Биржа, где торгуются углеродные квоты в Казахстане. Доступ только для крупных промышленных компаний. Банки не допущены. |
| **Compliance market (обязательный рынок)** | Рынок, где компании обязаны по закону покупать/продавать квоты. В КЗ: КазМунайГаз, ArcelorMittal, Казахмыс, KEGOC, угольные ТЭЦ. |
| **Voluntary market (добровольный рынок)** | Рынок, где компании покупают кредиты добровольно — для ESG-отчётности и улучшения имиджа. Kaspi, Halyk Bank, Air Astana, Shell. |
| **ESG** | Environmental, Social, Governance — критерии устойчивого развития. Компании публикуют ESG-отчёты, чтобы показать инвесторам и обществу свою «зелёность». |
| **Токенизация** | Процесс превращения реального актива (углеродный кредит) в цифровой токен на блокчейне. Кредит становится делимым, торгуемым и прозрачным. |
| **RWA (Real World Assets)** | Реальные активы, переведённые в блокчейн. Углеродные кредиты — типичный пример RWA. Хакатон Decentrathon 5.0, кейс 1 — именно про RWA. |
| **Double counting** | Когда одну и ту же тонну CO₂ засчитывают несколько раз. Например, один кредит заявлен и продавцом, и покупателем. Наша система решает это через сжигание (burn) — уничтоженный токен нельзя использовать повторно. |
| **Retire / гашение** | Процесс «использования» углеродного кредита. Компания заявляет: «Я компенсировала X тонн CO₂». Кредит при этом уничтожается навсегда. |
| **Верификация** | Процесс проверки реальности зелёного проекта. Независимый эксперт подтверждает, что проект действительно сокращает выбросы. |
| **MRV (Monitoring, Reporting, Verification)** | Система мониторинга, отчётности и верификации. Как именно мы доказываем, что проект работает. В будущем — IoT-датчики. |
| **Verra / Gold Standard** | Международные стандарты сертификации углеродных кредитов. В планах — интеграция с ними для выхода на мировой рынок. |
| **Жасыл Даму** | Государственная компания КЗ — оператор KZ ETS. Управляет реестром выбросов. Потенциальный партнёр. |

### Специфика CarbonKZ

| Термин | Что это простым языком |
|--------|----------------------|
| **KZTE** | Стейблкоин (крипто-валюта), привязанный 1:1 к казахстанскому тенге. 1 KZTE = 1 ₸. Используется для всех платежей на платформе вместо доллара или SOL. 6 десятичных знаков (1 KZTE = 1 000 000 минимальных единиц). |
| **CarbonToken** | Токен, представляющий 1 тонну CO₂. Создаётся для каждого проекта отдельно. Можно сжечь (retire) или передать. |
| **ShareToken** | Токен доли проекта. Покупая доли, вы получаете право на часть будущего дохода (дивиденды). Можно продать на маркетплейсе. |
| **Vault (хранилище)** | Специальный аккаунт (PDA), где хранятся все деньги (KZTE) проекта. Инвестиции попадают сюда, дивиденды выплачиваются отсюда. Никто, кроме программы, не может забрать деньги из vault. |
| **Дивиденды** | Доля дохода инвестора. Когда загрязнители покупают кредиты, выручка попадает в vault. Инвестор получает пропорционально своим долям. |
| **Escrow (эскроу)** | Механизм безопасной сделки. При выставлении долей на продажу (list_shares) токены «замораживаются» в эскроу. Покупатель платит → токены переходят к нему. Если продавец передумал → cancel_listing возвращает токены. |
| **RetireRecord** | Неизменяемая запись в блокчейне о гашении кредитов. Содержит: кто погасил, сколько тонн, когда, зачем. Это on-chain доказательство для аудиторов и ESG-отчётов. |
| **Faucet (кран)** | Раздатчик тестовых токенов. На devnet можно получить бесплатно KZTE и Carbon-токены для тестирования. Лимит: раз в 24 часа. |
| **Document Hash (doc_hash)** | SHA-256 хеш верификационного документа проекта. Записывается при создании и неизменяем. Верификатор должен предъявить документ с таким же хешем — это доказывает подлинность документа без его публикации. |
| **SHA-256** | Криптографическая хеш-функция. Превращает любой файл в уникальную строку из 64 символов. Если изменить хоть один байт файла — хеш будет совершенно другим. Как «цифровой отпечаток» документа. |
| **PRECISION** | Техническое число 10^12, используемое при расчёте дивидендов. Нужно для точности: сначала умножаем на PRECISION, потом делим. Без этого при делении маленьких сумм на большое число долей теряются копейки. |

## Аккаунты (данные в блокчейне)

Все данные проекта хранятся в 5 типах аккаунтов (PDA):

### CarbonProject — информация о зелёном проекте

| Поле | Тип | Описание |
|------|-----|----------|
| authority | PublicKey | Кошелёк владельца проекта |
| project_id | String (до 32 символов) | Уникальный ID проекта, например `"ses-yasavi"` |
| name | String (до 64 символов) | Название: «СЭС Университета Ахмеда Ясави» |
| project_type | Enum | Solar / Wind / Forest / Industrial / Other |
| carbon_mint | PublicKey | Адрес mint для CarbonToken этого проекта |
| share_mint | PublicKey | Адрес mint для ShareToken этого проекта |
| total_credits | u64 | Сколько тонн CO₂ всего может компенсировать проект |
| credits_retired | u64 | Сколько кредитов уже погашено (сожжено) |
| total_shares | u64 | Общее количество долей |
| shares_sold | u64 | Сколько долей уже продано |
| price_per_share | u64 | Цена одной доли в KZTE (в минимальных единицах) |
| vault | PublicKey | Адрес хранилища денег проекта |
| total_dividends_per_share | u128 | Накопленные дивиденды на одну долю (с множителем PRECISION) |
| document_hash | [u8; 32] | SHA-256 хеш верификационного документа |
| verified | bool | Прошёл ли проект верификацию |
| listing_count | u64 | Счётчик листингов (для уникальности PDA) |
| status | Enum | Active (принимает инвестиции) / Funded (все доли проданы) / Retired (завершён) |
| bump | u8 | Техническое число для PDA |

**Seeds:** `["project", project_id]`

| Поле | Тип | Описание |
|------|-----|----------|

| is_initialized | bool | Инициализирован ли аккаунт |
| bump | u8 | Техническое число для PDA |

**Seeds:** `["investor", адрес_проекта, адрес_кошелька]`

### VaultAccount — хранилище денег проекта

| Поле | Тип | Описание |
|------|-----|----------|

| total_deposited | u64 | Сколько всего KZTE поступило |
| total_claimed | u64 | Сколько KZTE было выплачено |
| bump | u8 | Техническое число для PDA |

## Формула дивидендов

Как рассчитываются дивиденды:

```
PRECISION = 1 000 000 000 000   (10^12, для точности вычислений)

Когда владелец вносит выручку (distribute_revenue):
   dividends_per_share += (сумма × PRECISION) / количество_проданных_долей

Когда инвестор забирает дивиденды (claim_dividends):
   невостребованное = (dividends_per_share - last_claimed) × мои_доли / PRECISION
```

**Пример:**
- Проект продал 1000 долей
- Владелец внёс 100 000 KZTE выручки
- dividends_per_share = 100 000 × 10^12 / 1000 = 100 × 10^12
- У вас 50 долей → ваши дивиденды = (100 × 10^12) × 50 / 10^12 = **5000 KZTE**

Важно: газ (комиссию сети) за получение дивидендов платит сам инвестор, а не владелец проекта. Это позволяет системе масштабироваться на любое количество инвесторов.
